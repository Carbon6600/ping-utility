# OSP Tools — ISP Ping Utility 📡

Проста, швидка та прозора утиліта на PowerShell для одночасного моніторингу стабільності двох інтернет-каналів (ISP-1 та ISP-2). 

Ідеально підходить для системних адміністраторів, мережевих інженерів або домашнього використання, коли потрібно «тут і зараз» перевірити якість лінків, переглянути статистику втрати пакетів або діагностувати миготіння мережі.

## ✨ Особливості

* 🚀 **Запуск в одну команду (One-Liner):** Не потрібно нічого завантажувати через браузер чи встановлювати. Скрипт виконується прямо з пам'яті.
* 🛡️ **Безпека (Без вірусів):** Код повністю відкритий (Open Source). Ви можете особисто перевірити кожен рядок перед запуском. Код валідується вбудованим Windows Defender (AMSI) прямо в ОЗУ.
* 👤 **Без прав адміністратора:** Скрипт працює в просторі користувача (User-space), не змінює системні файли, реєстр чи налаштування мережі.
* 📊 **Жива статистика:** Рахує загальну кількість запитів, кількість втрачених пакетів для кожного провайдера та виводить точний відсоток втрат (% Loss).
* 🔄 **Універсальність:** Працює на будь-якій версії Windows (PowerShell 5.1+ та PowerShell 7+).

---

## 🚀 Як запустити

Відкрийте звичайний **PowerShell** (**НЕ від імені адміністратора**) та виконайте одну з команд нижче.

### Варіант 1. Стандартний запуск (Google & Cloudflare)
Перевірка зв'язку за дефолтними резервними хостами (`8.8.4.4` та `1.0.0.1`):
```powershell
irm https://osp.pp.ua/ping | iex

```
### Варіант 2. Запуск із власними адресами (ISP / Хостами)
Якщо вам потрібно вказати конкретні IP-адреси локальних шлюзів, інтерфейсів або кастомні домени провайдерів, передайте їх як параметри:

```powershell
& { irm [http://osp.pp.ua/ping](http://osp.pp.ua/ping) | iex } -Target1 "192.168.88.1" -Target2 "my.isp.com"
```
Щоб зупинити роботу утиліти та припинити опитування, натисніть комбінацію клавіш **Ctrl + C**.

Приклад виводу в консолі
